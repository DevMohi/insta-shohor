FAQ

To exactly know how js works, we have to go through programs that a browser executes before running the js code. Firstly, the DOM parser converts the html to a strucure which we visually see, second the CSS parser takes the CSS code and make sure the document is rendered correctly, both these process combined is one program and is known as rendering engine. Lastly, since JS is an interpreted language, therefore the code is passed through a compliler.Most modern browsers use a technology also known as JIT(Just-In-time) its job is to take human readable instructions and convert it to machine code ,used in each browser,also it optimises the code.

Js depends on a runtime model known as event loop, it is in charge of executing the code, gathering and processing events, and executing queued sub-taks. Now lets describe how it works, what is does is it observes the Call Stack and the Callback Queue, So , if the Call Stack is empty , it will bring the first event from the queue and push it to the Call Stack, which then runs it. Such a looping is called a tick in the Event loop.

The similarities between local storage and session storage is huge, both are used to store data in the browser. They both are accessible from the client side only and since they are a part of front-end only, they have no SSL support. They Both sit in your browser until someone requests it. Local storage has a limit of 5mb, and Session storage around 5-10mb limit. Now , the main difference between them is that local storage has no expiration date while Session Storage data are gone when you close the browser tab - thus it is called session.
